快速入口

调试时最常用的三个接口。对外统一建议走 POST /transcribe；图片 URL 会自动识别，也可以显式传 task=image。/ocr 仅保留兼容。当前用量查询针对腾讯 ASR。账号选择策略是“优先剩余小时最多的账号”，并带 3 分钟的短时预扣减，避免短时间内一直打同一个号。

GET  /health
GET  /tencent/quota
POST /transcribe
curl --noproxy '*' -s 'http://127.0.0.1:8014/tencent/quota'

curl --noproxy '*' -s \
  'http://127.0.0.1:8014/tencent/quota?start_date=2026-04-01&end_date=2026-04-21&biz_names=asr_rec,asr_rt'
如果想强制刷新腾讯最新用量：

curl --noproxy '*' -s \
  'http://127.0.0.1:8014/tencent/quota?refresh=1'

使用文档

把接口使用和账号注册配置集中到一个页面里，便于交接和部署。

API 文档

1. 健康检查：GET /health

2. 用量查询：GET /tencent/quota

3. 通用转文字：POST /transcribe（音频 / 图片统一入口）

curl --noproxy '*' -X POST 'http://127.0.0.1:8014/transcribe' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://example.com/demo.mp3",
    "raw": true
  }'
curl --noproxy '*' -X POST 'http://127.0.0.1:8014/transcribe' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://example.com/demo.jpg",
    "task": "image",
    "raw": true
  }'

配置注册文档

1. 从模板复制配置：

cp transcribe_config.template.json transcribe_config.json

2. 在 asr.tencent.accounts 里配置多个腾讯账号，语音转写和图片 OCR 都会复用这组腾讯密钥。

3. 每个账号建议填写。你现在这类测试号如果是“每月免费 10 小时”，就填 monthly_quota_seconds: 36000：

{
  "name": "account-1",
  "secret_id": "AKIDxxxxx",
  "secret_key": "xxxxx",
  "region": "ap-beijing",
  "monthly_quota_seconds": 36000
}

4. 服务默认会按账号轮询；若请求里显式传了单次密钥，则单次密钥优先。

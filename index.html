快速入口

右侧不再堆大段说明，改成短说明加命令块。调试时优先看健康检查、额度查询和统一转写入口。

健康检查

确认服务是否在线，适合部署后第一时间验证。

额度查询

查看腾讯 ASR 用量、剩余额度、账号选择策略和 OCR 调用统计。

统一转写

音频和图片统一走 POST /transcribe，图片可自动识别，也可显式传 task=image。如需限制账号范围，可传 allow 白名单数组。

账号申请

新腾讯账号先走 /apply 提交申请，审核通过后才写入配置并热加载。

审核撤销

管理员在 /review 验证密钥、通过、拒绝，或撤销最近一次已生效审批。

选择策略

当前默认优先剩余小时最多的账号，并带 3 分钟短时预扣减，避免短时间请求持续压到同一个腾讯号。

强制刷新用量

参数refresh=1
用途跳过短时缓存，拉取最新腾讯用量
curl --noproxy '*' -s \
  'http://127.0.0.1:8014/tencent/quota?refresh=1'

使用文档

把接口使用和账号注册配置集中到一个页面里，便于交接和部署。

API 文档

  • 健康检查：GET /health
  • 用量查询：GET /tencent/quota
  • 申请列表：GET /tencent/account-requests（管理员口令）
  • 密钥验证：POST /tencent/account-credentials/validate（管理员口令）
  • 提交申请：POST /tencent/account-requests
  • 通用转文字：POST /transcribe（音频 / 图片统一入口）
  • 白名单负载：请求体可传 allow: ["account-1","account-2"]，只在指定账号内选取。
curl --noproxy '*' -X POST 'http://127.0.0.1:8014/transcribe' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://example.com/demo.mp3",
    "raw": true
  }'
curl --noproxy '*' -X POST 'http://127.0.0.1:8014/transcribe' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://example.com/demo.mp3",
    "asr_provider": "tencent",
    "allow": ["account-1", "account-2"],
    "raw": true
  }'
curl --noproxy '*' -X POST 'http://127.0.0.1:8014/transcribe' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://example.com/demo.jpg",
    "task": "image",
    "raw": true
  }'

配置注册文档

  • 从模板复制配置文件。
  • 在 asr.tencent.accounts 中配置多个腾讯账号，语音转写和图片 OCR 会复用这组密钥。
  • 免费测试号如果是“每月免费 10 小时”，填 monthly_quota_seconds: 36000。
  • 服务默认在账号池内负载；如果请求里带 allow 白名单，则只在白名单账号里负载。
  • 如果请求里带单次密钥（tencent_secret_id / tencent_secret_key），则单次密钥优先。
  • 审批数据单独保存在本地 JSON；审核通过后才会改 transcribe_config.json。
cp transcribe_config.template.json transcribe_config.json
{
  "name": "account-1",
  "secret_id": "AKIDxxxxx",
  "secret_key": "xxxxx",
  "region": "ap-beijing",
  "monthly_quota_seconds": 36000
}
